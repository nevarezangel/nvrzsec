Static Code Analysis

This write-up covers the basics and tools involved in static code analysis, a technique for finding security vulnerabilities without executing the code.

Overview

Static code analysis is performed by examining source code or binaries to detect issues like:

  • Hardcoded credentials
  • Unvalidated input
  • Unsafe functions
  • Potential logic flaws

Tools Used

  • Bandit (Python)
  • SonarQube
  • Semgrep
  • Checkmarx / Fortify

Example Output

>>> bandit myscript.py
[bandit]	INFO	running on Python 3.9.2
>> Issue: Use of insecure MD5 hash function.
   Severity: Medium   Confidence: High
   Location: myscript.py:24
